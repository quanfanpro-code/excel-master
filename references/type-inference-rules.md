# 类型推断规则

## 优先级

pct > date > text > money > 值采样

先跑关键词匹配，全部不命中再走值采样。

## 关键词匹配表

| 类型 | 关键词（正则） | 示例 |
|------|----------------|------|
| pct | `率$`、`占比`、`百分比`、`增长率`、`rate$`、`ratio$` | 毛利率、完成率、增长率、占比、win_rate |
| date | `日期`、`时间`、`年月`、`期间`、`年份`、`月份`、`date`、`time`、`year`、`month` | 下单日期、创建时间、create_date、report_year |
| text | `编号`、`id`、`单号`、`编码`、`电话`、`手机`、`备注`、`说明`、`名称`、`地址`、`描述`、`号码`、`订单号`、`负责人`、`姓名`、`联系人`、`部门`、`岗位`、`phone`、`email`、`code`、`desc`、`address` | 商品名称、订单号、负责人、phone_number、email_address、product_code |
| money | `金额`、`价格`、`收入`、`成本`、`费用`、`毛利额`、`净利`、`合计`、`总额`、`售价`、`单价`、`预算`、`支出`、`毛利(?!率)`、`amount`、`price`、`cost`、`revenue`、`total`、`budget`、`expense` | 预算、金额、单价、毛利额、total_amount、total_revenue |

**重要：** `率$` 必须在 money 关键词之前匹配。列名"毛利率"含"毛利"但不含"率$"会误判为 money。当前用 `率$` 限定以"率"结尾的词先走 pct 分支，再用 `毛利(?!率)` 排除"毛利率"式误匹配。

## 值采样逻辑

关键词未命中时，取前 20 个非空值采样：

1. **datetime 类型** → date
2. **数值列**：
   - float + 全在 0~1 之间 + 唯一值 > 2 → pct
   - float + 不符合上述条件 → money
   - int → number
3. **字符串列**：检查是否有 12 位以上纯数字 → text
4. **无法判断** → unknown（走 `@` 文本格式）

## 常见误判案例

| 列名 | 关键词匹配 | 实际类别 | 处理方式 |
|------|-----------|----------|----------|
| 毛利率 | 含"毛利"→走 money ❌ | pct | `率$` 优先匹配修正 |
| 负责人 | 无匹配 → unknown | text | 加入 text 关键词列表 |
| 单价 | 含"单价"→走 money ✅ | money | 正确 |
| 完成率 | 含"率$"→走 pct ✅ | pct | 正确 |
| 数量 | 无匹配 → int→number ✅ | number | 正确 |
